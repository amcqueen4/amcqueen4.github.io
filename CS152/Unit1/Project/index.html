Code Golfing

Written by Andrew J. McQueen

What is it?

Code golfing can be viewed as a competitive genre of coding. The goal, similar to golf, is to have the lowest score (characters in this case). The priority is not readability of the code, but instead is the compression of the code.

Why do it?

For fun! The entire point is to get better at problem solving. It can require some serious work to understand to find the shortest solution to a problem in the language you are working with and being able to do so shows serious dedication with the language you are working with.

How can I do it?

There are plenty of ways you can do it. To start you will want to become exremely familiar with a language that you are working with. Golfing often requires hacky and messy tricks often overlooked or obscure in general.

Good languages for golfing, due to their simplicity or liberty in syntax include:

  1. Ruby
    • Extremely flexible syntax
    • Lots of convenient features to abstract complexity
    • Super easy to learn
    • A very powerful golfing language.
  2. Lua
    • Built to be very easy to learn
    • Quirky features such as metatables
    • Generous abstraction
  3. Python
    • Often many people's first language due to how simple it is.
    • Popular in code golfing.

Any Examples?

Yes! I do have a set of examples that I like showing.

The Basic Factorial in Ruby

The factorial (!) of a number just represents the product sum of all whole numbers greater than zero below the given number. That is to say: 5! (5 factorial) = 5 * 4 * 3 * 2 * 1 = 120

In ruby you can easily express the factorial of an inputted number by using a range and then calling a reducer method on it. 

    (1..gets.to_i).inject :*

Short Function Declarations in Lua

In lua you can define a function very simply. 
    function funcName(paramA, paramB, ...)
    -- function body
end
We can use the fact that lua has a "load" and "loadstring" method. "load" is shorter so we will use that. This allows us to define lua code as a string in lua and execute arbitrary code. We can abuse varadic parameter assignment to pass parameters to our function. Here is an example that could be used to just print all characters in a string. 
    ("hello"):gsub(".",load"print(...)")
This is a poor example though because if you wanted to just print out the characters in a string then you would just pass the "print" function itself rather than wrapping it in a function, but there are real reasons to do use "load" 
    ("hello"):gsub(".",print)

Abuse of Ternary for Control Flow in Lua

Abuse of ternary is very useful when you are golfing in Lua. Writing an "if else" statement is too long, so it becomes dire to find a shorter way to do two different things under different cases.

Because of how Lua works, if the condition in an statement is true then further conditions do not need to be evaluated. This brings up interesting calling syntax.

Below we have a useless variable assignment and upon "condition" evaluating to true the function "f" is called, else "g" is called. "g()" is never evaluated if "f()" is done, and vice versa. 

    _=condition and f()or g()
